# Job Application Tracker — Build Spec

Build a local-first job application tracker with two components: a FastAPI backend with a web UI, and a JavaScript bookmarklet that scrapes job postings and sends them to the backend.

## Stack
- **Backend:** Python 3.11+, FastAPI, SQLAlchemy, SQLite, Uvicorn
- **Frontend:** Server-rendered HTML via Jinja2 templates + HTMX for interactivity (no build step, no React)
- **Bookmarklet:** Vanilla JS, single-line minified, generated by a build script

## Database schema
Single `jobs` table:
- `id` (int, PK)
- `company` (str, required)
- `role` (str, required)
- `location` (str, nullable)
- `url` (str, required, unique — dedupe by URL)
- `source` (str — which ATS: "greenhouse", "lever", "ashby", "workable", "workday", "other")
- `status` (str, enum: "interested", "applied", "phone_screen", "onsite", "offer", "rejected", "withdrawn", default "interested")
- `notes` (text, nullable)
- `created_at` (datetime, default now)
- `updated_at` (datetime, auto-updated on change)

On duplicate URL POST, return the existing row instead of erroring.

## Backend endpoints
- `GET /` — render `index.html` with all jobs, sorted by `updated_at` desc, grouped or filterable by status
- `POST /api/jobs` — accept JSON `{company, role, location, url, source}`, create or return existing, respond with the job as JSON. **Must have permissive CORS** (`allow_origins=["*"]`) so the bookmarklet works from any ATS domain.
- `PATCH /api/jobs/{id}` — update status, notes, or any field. Used by HTMX from the UI.
- `DELETE /api/jobs/{id}` — delete a job, returns empty 200 for HTMX swap.
- `GET /api/jobs/{id}` — return a single job (used to refresh a row).

## Frontend UI (index.html)
- Header with total count and a count per status
- Filter bar: "All / Interested / Applied / Phone Screen / Onsite / Offer / Rejected"
- Table with columns: Company | Role | Location | Status (inline-editable dropdown) | Source | Date Added | Notes (expandable) | Delete
- Status dropdown uses HTMX to PATCH inline — no full page reload
- Notes column: click to expand into a textarea, blur-to-save via HTMX
- Clean, minimal styling. Use system fonts, generous spacing, subtle borders. No CSS framework — just hand-written CSS in `static/styles.css`. Dark-mode-aware via `prefers-color-scheme`.

## Bookmarklet (`bookmarklet/src.js`)

The bookmarklet must:
1. Detect which ATS the current page is on by checking `window.location.hostname` and URL patterns
2. Run the appropriate scraper to extract `{company, role, location, url, source}`
3. POST to `http://localhost:8000/api/jobs`
4. Show a small toast notification (injected DIV, fixed position top-right, auto-dismisses after 3s) — green on success, red on error, includes the company + role text

### ATS detection and scraping rules

**Greenhouse** (`boards.greenhouse.io`, `job-boards.greenhouse.io`, or `*.greenhouse.io` embedded):
- Company: from URL path `/{company}/jobs/...` or `.company-name` element
- Role: `h1.app-title` or `h1`
- Location: `.location` or `.job__location`

**Lever** (`jobs.lever.co`):
- Company: from URL path `jobs.lever.co/{company}/...`
- Role: `.posting-headline h2` or `h2`
- Location: `.posting-categories .location` or `.location`

**Ashby** (`jobs.ashbyhq.com`):
- Company: from URL path `jobs.ashbyhq.com/{company}/...`
- Role: `h1` or `[class*="JobPostingTitle"]`
- Location: `[class*="JobPostingLocation"]` or near the role

**Workable** (`apply.workable.com`, `jobs.workable.com`):
- Company: from URL path or `[data-ui="company-name"]`
- Role: `h1[data-ui="job-title"]` or `h1`
- Location: `[data-ui="job-location"]`

**Workday** (`*.myworkdayjobs.com`):
- Best-effort: company from subdomain (e.g., `nvidia.wd5.myworkdayjobs.com` → "nvidia")
- Role: `[data-automation-id="jobPostingHeader"]` or `h1` or `h2`
- Location: `[data-automation-id="locations"]` or `[data-automation-id="locationData"]`
- If scraping fails partially, send what you have — backend accepts nullable location

**Fallback (other)**:
- Company: hostname without TLD/www
- Role: `document.title` (cleaned of common suffixes like " - Careers", " | Jobs")
- Location: null
- Source: "other"

For all scrapers: trim whitespace, fall back to `null` for missing fields except company/role/url which should always have *something* (use placeholders like "Unknown" if truly nothing found, so the user can fix it in the UI).

### Build script (`bookmarklet/build.py`)
Reads `src.js`, wraps it in `javascript:(function(){...})();`, URL-encodes it, prints it to stdout. User copies the output and creates a browser bookmark with that as the URL.

## README.md
Include:
- Setup: `pip install -r requirements.txt`, `uvicorn backend.main:app --reload`
- How to generate and install the bookmarklet (run `python bookmarklet/build.py`, copy output, create bookmark)
- How to add support for a new ATS (edit `src.js`, rebuild)
- Troubleshooting CORS / mixed content (note: some HTTPS sites may block requests to http://localhost — if so, point at the browser's setting or suggest running backend with a self-signed cert)
